# YouTube API Quota Review Screencast Guide

To obtain your YouTube API Quota increase, the YouTube compliance reviewers need a video demonstration (screen-cast) showing that your application uses the requested API scopes legitimately. 

Follow these steps to record a screen-cast that will be approved on the first attempt.

---

## 🎬 Prep Work (Before Recording)
1. **Choose a Screen Recorder**: Use OBS Studio, Windows Game Bar (`Win + G`), Zoom (share screen and record), or any recording software of your choice. Ensure your resolution is clear and the terminal text is legible.
   > [!NOTE]
   > **Face recording is NOT required.** They only need to see your computer screen, your code/terminal window, your browser during authorization, and your YouTube Studio dashboard.
2. **Clear Open Tabs**: Open a clean browser window. Open a terminal panel (PowerShell or Command Prompt).
3. **Verify Files**: Ensure `config/youtube_client_secrets.json` is present in your project.
   > [!NOTE]
   > **What is `youtube_client_secrets.json`?** This is the OAuth 2.0 credential file containing your Client ID and Client Secret downloaded from the Google Cloud Console for the `autoreel-empire` project. It is already present in your `config` folder and is used to launch the Google login screen.
4. **Locate the Channel Link**: Find the link to your channel (e.g., `Example Tech AI` or whichever channel you are uploading to). Have this link ready to paste in your email reply.

---

## 🎥 Recording Script & Steps

### Step 1: Start Recording (0:00 - 0:15)
*   Start the recording showing your clean desktop, code editor, or terminal.
*   *Action*: Highlight the project structure, explaining or demonstrating that this is a private, locally-run Python video automation project. (No face camera needed!)

### Step 2: Show the OAuth Consent & Scope Screen (0:15 - 0:45)
*   *Action*: Open your terminal and run the command to navigate to the project directory:
    ```cmd
    cd c:\Users\dell\Desktop\autoReel\autoReel
    ```
*   *Action*: Run the authorization script:
    ```bash
    python execution/authorize_youtube.py
    ```
*   *Action*: Select your channel profile (e.g., type `example_tech_ai`).
*   *Action*: The browser will automatically open showing the Google login screen. Log in to the Google account associated with the channel.
*   *Action*: **CRITICAL**: Expand the "Advanced" options if you see "Google hasn't verified this app" and click **Go to AutoReel (unsafe)**. Show the permissions consent list. Ensure the recording captures the specific scopes requested:
    - **Manage your YouTube videos** (`youtube.upload` and `youtube`)
    - **View YouTube Analytics reports** (`yt-analytics.readonly` and `youtube.readonly`)
*   *Action*: Click **Continue** to approve the permissions. Show the successful message in the browser: *"The authentication flow has completed. You may close this window."*
*   *Action*: Show the terminal output confirming: `✅ Authorization complete! Token saved to channels/...`

### Step 3: Run the Upload Test Script (0:45 - 1:15)
*   *Action*: In your terminal, run the compliance test upload script:
    ```bash
    python execution/compliance_upload_test.py
    ```
*   *Action*: Select your authorized channel (press Enter to accept the default `example_tech_ai`).
*   *Action*: Watch the script run. The terminal will log:
    - Generating the dummy media assets (Pillow generating thumbnail, FFmpeg rendering a 3s blue video).
    - Connecting to the YouTube Data API.
    - Executing `youtube.videos().insert()` showing progress bar `0%` to `100%`.
    - Outputting the generated video ID.
    - Executing `youtube.thumbnails().set()` to bind the custom thumbnail.
    - Printing the final Watch URL.

### Step 4: Verify the Upload in YouTube Studio (1:15 - 1:45)
*   *Action*: Click the watch URL printed in the terminal (or copy-paste it into your browser). Show the video playing.
*   *Action*: Open **YouTube Studio** (`studio.youtube.com`), go to the **Content** tab.
*   *Action*: Point out the uploaded test video. Show that:
    1. It is marked as **Unlisted** (which complies with the script parameters).
    2. It has the **Custom Thumbnail** set correctly (blue verification screen).
    3. The title and description match the automated metadata generated by the script.

### Step 5: Stop Recording & Save
*   End the recording. Keep the video length under 2 minutes.
*   Upload the video file to Google Drive or YouTube (as unlisted), and copy the sharing link to send in your email response.
